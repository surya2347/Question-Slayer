# Bloom 스코어링 프롬프트 설계 계획

## 📋 개요
- **목표**: 사용자 질문을 분석해 Bloom 분류학 6단계 중 어디에 해당하는지 판별하고 JSON으로 반환
- **이론**: Bloom(1956) 인지적 영역 원년 6단계 기준
- **방식**: 키워드 1차 판별 → LLM 2차 정교화 (혼합)
- **위치**: `core/prompts.py` (스코어링 프롬프트) + `core/utils.py` (Bloom 레이블 매핑)
- **활용**: `core/graph.py`의 `score_bloom()` 노드에서 호출

---

## 📚 Bloom 1956년 6단계 정의

| 단계 | 영문 | 한국어 | 핵심 정의 |
|-----|------|--------|---------|
| 1 | Knowledge | 지식 | 사실, 용어, 기본 개념을 **기억**하는 능력 (의미 이해 불필요) |
| 2 | Comprehension | 이해 | 사실과 개념을 **이해**하고 요약·정리하는 능력 |
| 3 | Application | 응용 | 습득한 지식으로 새로운 상황에서 **문제를 해결**하는 능력 |
| 4 | Analysis | 분석 | 정보를 나눠 **관계, 동기, 원인을 파악**하는 능력 |
| 5 | Synthesis | 종합 | 여러 요소를 **결합해 새로운 전체를 창출**하는 능력 |
| 6 | Evaluation | 평가 | 기준·표준에 따라 정보에 대해 **판단**을 내리는 능력 |

---

## 🎯 판별 전략: 혼합 방식

```
[사용자 질문]
     ↓
1차: 키워드 매칭 (빠름, 무료)
     ├─ 신뢰도 HIGH (0.8 이상) → 바로 결과 반환
     └─ 신뢰도 LOW (0.8 미만) → 2차로 넘김
                                      ↓
                             2차: LLM 분석 (정확, 유료)
                                      ↓
                             JSON 결과 반환
```

## 🔑 1차: 키워드 판별 설계

### 단계별 트리거 키워드

**Level 1 (지식) — "무엇?", "언제?", "누가?"**
```
- 의문사: 무엇, 뭐, 누가, 언제, 어디서, 몇 개, 몇 년
- 패턴: "~란?", "~이란?", "~의 정의", "~이 무엇", "~을 나열", "~의 뜻"
- 예시: "Python 변수란?", "NCS의 정의가 뭔가요?", "데이터베이스가 뭐예요?"
```

**Level 2 (이해) — "어떻게 작동?", "왜?", "설명해주세요"**
```
- 패턴: "어떻게 작동", "왜 ~인가", "~를 설명", "~의 의미", "~는 어떤"
- 예시: "변수가 어떻게 작동하나요?", "왜 포인터를 쓰나요?", "TCP/IP를 설명해주세요"
```

**Level 3 (응용) — "어떻게 사용?", "활용", "적용하면?"**
```
- 패턴: "어떻게 사용", "~를 활용", "~에 적용", "써보면", "~로 만들면"
- 예시: "변수를 어떻게 활용하나요?", "이 개념을 실무에 적용하면?"
```

**Level 4 (분석) — "차이점?", "비교", "왜 이렇게?"**
```
- 패턴: "차이점", "비교하면", "왜 ~하나", "어떤 구조", "분류하면", "나누면"
- 예시: "TCP와 UDP의 차이점은?", "왜 객체지향을 쓰나요?"
```

**Level 5 (종합) — "결합", "설계", "새로운 방법"**
```
- 패턴: "결합하면", "설계해보면", "새로운 ~", "만들어보면", "통합하면"
- 예시: "REST API와 GraphQL을 결합한 아키텍처를 설계한다면?"
```

**Level 6 (평가) — "평가", "어느 것이 더 좋은가?", "판단"**
```
- 패턴: "평가하면", "더 나은", "판단", "장단점", "최선의 방법", "추천해주세요"
- 예시: "Python과 JavaScript 중 어느 것이 더 좋은가요?", "이 코드를 평가해주세요"
```

### 신뢰도(confidence) 계산
```python
# 매칭된 키워드 수에 따라
1개 매칭 → confidence = 0.6
2개 매칭 → confidence = 0.75
3개 이상 → confidence = 0.9

# 신뢰도 임계값
>= 0.8 → 1차에서 바로 반환 (LLM 불필요)
< 0.8  → 2차 LLM 정교화 실행
키워드 없음 → 2차 LLM 정교화 실행
```

## 🤖 2차: LLM 정교화 프롬프트 설계

### 프롬프트 구조

```python
PROMPT_BLOOM_SCORING = """
당신은 Bloom의 교육목표 분류학(1956) 전문가입니다.

【분석할 질문】
{question}

【Bloom 분류학 6단계 기준】
1. 지식(Knowledge): 사실, 용어, 개념을 기억/인식하는 능력
   - 키워드: 무엇, 정의, 나열, 기억, 언제, 누가
2. 이해(Comprehension): 정보를 이해하고 요약하는 능력
   - 키워드: 설명, 왜, 어떻게 작동, 의미, 해석
3. 응용(Application): 지식을 새 상황에 적용하는 능력
   - 키워드: 활용, 적용, 사용하면, 실무에서, 어떻게 쓰나
4. 분석(Analysis): 정보를 분해해 관계/원인을 파악하는 능력
   - 키워드: 차이점, 비교, 왜 이렇게, 구조, 분류
5. 종합(Synthesis): 요소들을 결합해 새것을 창출하는 능력
   - 키워드: 결합, 설계, 새로운 방법, 통합, 만들어보면
6. 평가(Evaluation): 기준에 따라 판단하는 능력
   - 키워드: 평가, 더 나은, 장단점, 판단, 최선, 추천

【지시사항】
1. 위 질문이 Bloom 6단계 중 어디에 해당하는지 분석하세요.
2. 어떤 단어/표현이 그 단계를 나타내는지 근거를 제시하세요.
3. 반드시 JSON 형식으로만 응답하세요 (다른 텍스트 포함 금지).

【필수 JSON 형식】
{{
    "level": [1~6 정수],
    "name_ko": "[지식|이해|응용|분석|종합|평가]",
    "name_en": "[Knowledge|Comprehension|Application|Analysis|Synthesis|Evaluation]",
    "definition": "[해당 단계의 핵심 정의 한 문장]",
    "evidence": "[이 질문이 해당 단계인 이유 1~2문장]",
    "keywords_found": ["[질문에서 발견된 핵심 키워드 목록]"],
    "confidence": [0.0~1.0 소수]
}}
"""
```

### 응답 JSON 스키마

```json
{
    "level": 3,
    "name_ko": "응용",
    "name_en": "Application",
    "definition": "습득한 지식을 새로운 상황에서 문제를 해결하는 능력",
    "evidence": "질문에서 '어떻게 활용하나요?'라는 표현이 새로운 상황에 지식을 적용하는 응용 수준을 나타냅니다.",
    "keywords_found": ["활용", "어떻게"],
    "confidence": 0.87
}
```

## 🔧 최종 반환 JSON 설계

혼합 방식 최종 결과 통합 스키마:

```json
{
    "level": 3,
    "name_ko": "응용",
    "name_en": "Application",
    "definition": "습득한 지식을 새로운 상황에서 문제를 해결하는 능력",
    "evidence": "질문에서 '활용', '사용' 키워드가 발견되었습니다.",
    "keywords_found": ["활용", "사용"],
    "confidence": 0.87,
    "method": "keyword",
    "score": 3,
    "max_score": 6,
    "feedback": "현재 응용(3단계) 수준의 질문입니다. 분석(4단계)로 올라가려면 '왜 이렇게 되는가?', '차이점은?' 같은 질문을 시도해보세요."
}
```

| 필드 | 타입 | 설명 |
|-----|------|-----|
| `level` | int | Bloom 단계 번호 (1~6) |
| `name_ko` | str | 단계 한국어 이름 |
| `name_en` | str | 단계 영문 이름 |
| `definition` | str | 해당 단계 정의 |
| `evidence` | str | 이 질문이 해당 단계인 근거 |
| `keywords_found` | list | 판별에 사용된 키워드 |
| `confidence` | float | 판별 신뢰도 (0.0~1.0) |
| `method` | str | 사용된 판별 방식 (keyword / llm / hybrid) |
| `score` | int | 단계 점수 (level과 동일) |
| `max_score` | int | 최고 점수 (항상 6) |
| `feedback` | str | 다음 단계로 올라가기 위한 가이드 |
