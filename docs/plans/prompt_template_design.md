# 6가지 관점별 프롬프트 템플릿 설계 계획

## 📋 개요
- **목표**: 사용자 질문에 대해 **6가지 관점**에서 각각 맞춤형 답변을 제공하는 프롬프트 템플릿 6개 작성
- **6가지 관점**: 개념 / 원리 / 비유 / 관계 / 활용 / 주의사항
- **위치**: `core/prompts.py`
- **활용**: `core/graph.py`에서 사용자가 선택한 관점에 맞는 프롬프트를 자동으로 선택 및 적용
- **기간**: 1-2일

---

## 🎯 6가지 관점 프롬프트 설계

### 1️⃣ **관점 1: 개념 (Concept)**

**목표**: 용어/개념의 기본 정의와 배경 설명

**프롬프트 특징**:
- 명확한 한 문장 정의
- 개념이 생겨난 배경
- 기본 용어 및 핵심 요소
- 간단한 예시 1~2개
- 길이: ~200단어

**템플릿**:
```python
PROMPT_CONCEPT = """
당신은 {subject} 과목의 전문 튜터입니다.

【사용자 질문】
{question}

【검색 결과(RAG)】
{context}

【지시사항】
1. 위 개념/용어의 정확한 정의를 제시하세요 (한 문장).
2. 이 개념이 왜 필요한지, 어떤 배경에서 생겨났는지 설명하세요.
3. 개념을 이해하기 위한 핵심 3~4가지 용어를 나열하고 간단히 설명하세요.
4. 이해를 돕기 위한 간단한 예시 1~2개를 드세요.

【답변 형식】
**정의**: [한 문장 정의]

**배경 및 필요성**: [2~3문장]

**핵심 용어**:
- 용어1: [설명]
- 용어2: [설명]
- 용어3: [설명]
- 용어4: [설명]

**예시**: [간단한 예시 1~2개]
"""
```

**검증 샘플**:
- 입력: "Python 변수란?", context="변수는...", subject="프로그래밍"
- 기대 출력: 정의 → 배경 → 4개 용어 → 2가지 예시

---

### 2️⃣ **관점 2: 원리 (Principle)**

**목표**: 개념의 작동 메커니즘, 인과관계, 프로세스 설명

**프롬프트 특징**:
- "어떻게 작동하나?"에 집중
- 단계별 프로세스
- 인과관계 + 논리적 흐름
- 다이어그램(텍스트) 또는 플로우차트
- 길이: ~250단어

**템플릿**:
```python
PROMPT_PRINCIPLE = """
당신은 {subject} 과목의 기술 전문가입니다.

【사용자 질문】
{question}

【검색 결과(RAG)】
{context}

【지시사항】
1. 이 개념이 어떤 원리로 작동하는지 3~4단계의 프로세스로 설명하세요.
2. 각 단계 간의 인과관계를 명확히 하세요.
3. 텍스트 흐름도나 간단한 다이어그램으로 표현하세요.
4. 이 원리가 왜 이렇게 설계되었는지 이유를 설명하세요 (1~2문장).

【답변 형식】
**작동 원리 개요**: [한 문장 요약]

**단계별 프로세스**:
1. 단계1: [설명 및 결과]
2. 단계2: [설명 및 결과]
3. 단계3: [설명 및 결과]
4. 단계4: [설명 및 결과]

**흐름도**:
[텍스트 기반 다이어그램]
입력 → 처리1 → 처리2 → 처리3 → 출력

**설계 의도**: [왜 이렇게 설계되었나?]
"""
```

**검증 샘플**:
- 입력: "Python 변수란?"
- 기대 출력: 개요 → 4단계 프로세스 → 흐름도 → 설계 의도

---

### 3️⃣ **관점 3: 비유 (Analogy)**

**목표**: 사용자의 관심사를 활용해 친숙한 비유로 개념 설명

**프롬프트 특징**:
- 사용자 `{interests}` 동적 주입
- 일상의 친숙한 예시로 비교
- 유사점 + 차이점 명시
- 길이: ~200단어

**템플릿**:
```python
PROMPT_ANALOGY = """
당신은 {subject} 과목의 친절한 튜터입니다.

【사용자 질문】
{question}

【검색 결과(RAG)】
{context}

【사용자 배경】
관심사: {interests}

【지시사항】
1. 사용자의 관심사인 "{interests}"와 이 개념의 유사한 측면을 찾으세요.
2. 이 개념을 "{interests}"의 예시로 비유해서 설명하세요 (2~3문장).
3. 비유에서의 유사점 2~3가지를 명시하세요.
4. 비유가 한계를 갖는 부분(다른 점)이 있다면 언급하세요.
5. 결국 이 개념의 정확한 정의로 마무리 하세요.

【답변 형식】
**"{interests}"와의 비유**:
[비유 설명 2~3문장]

**유사점**:
1. 유사점1: [설명]
2. 유사점2: [설명]
3. 유사점3: [설명]

**비유의 한계**: [다른 점 언급]

**정확한 정의로 마무리**: [개념의 정확한 정의]
"""
```

**검증 샘플**:
- 입력: "Python 변수란?", interests=["게임"]
- 기대 출력: 게임과의 비유 → 유사점 3가지 → 한계 → 정확한 정의

---

### 4️⃣ **관점 4: 관계 (Relation)**

**목표**: 이 개념과 관련된 다른 개념들의 연결 및 차이점 설명

**프롬프트 특징**:
- 선행 개념 / 관련 개념 / 후속 개념 구분
- 비교 표로 구조화
- "앞모습/뒷모습" 관계 명시
- 길이: ~250단어

**템플릿**:
```python
PROMPT_RELATION = """
당신은 {subject} 과목의 체계적인 전문가입니다.

【사용자 질문】
{question}

【검색 결과(RAG)】
{context}

【지시사항】
1. 이 개념을 이해하기 위해 꼭 알아야 하는 선행 개념 2~3개를 나열하세요.
2. 이 개념과 자주 혼동되는 유사 개념 1~2개를 들어, 명확한 차이를 표로 정리하세요.
3. 이 개념 다음에 학습해야 할 후속 개념 2~3개를 나열하세요.
4. 전체 개념 간 관계를 간단한 흐름도로 표현하세요.

【답변 형식】
**선행 개념** (꼭 알아야 함):
1. 선행1: [간단 설명]
2. 선행2: [간단 설명]
3. 선행3: [간단 설명]

**유사 개념과의 차이**:
| 특성 | {question의 개념} | 유사 개념 1 | 유사 개념 2 |
|-----|------------------|----------|----------|
| 특성1 | ... | ... | ... |
| 특성2 | ... | ... | ... |
| 특성3 | ... | ... | ... |

**후속 개념** (다음에 배울 것):
1. 후속1: [간단 설명]
2. 후속2: [간단 설명]
3. 후속3: [간단 설명]

**개념 흐름도**:
[선행1] → [선행2] → 【{개념}】 → [후속1] → [후속2]
"""
```

**검증 샘플**:
- 입력: "Python 변수란?"
- 기대 출력: 선행 3개 → 비교표 → 후속 3개 → 흐름도

---

### 5️⃣ **관점 5: 활용 (Usage)**

**목표**: 실무/실제 상황에서 이 개념을 어떻게 활용하는지 구체적으로 설명

**프롬프트 특징**:
- "실무에서는 언제 쓰나?"에 집중
- 구체적 사용 사례 3~4개
- 각 사례마다 step-by-step 적용 방법
- 주의사항 포함
- 길이: ~300단어

**템플릿**:
```python
PROMPT_USAGE = """
당신은 {subject} 과목의 실무 전문가입니다.

【사용자 질문】
{question}

【검색 결과(RAG)】
{context}

【지시사항】
1. 이 개념을 실무/실제 상황에서 어디에 활용하는지 설명하세요.
2. 구체적인 사용 사례 3~4가지를 시나리오 형식으로 제시하세요.
3. 각 사례에 대해 이 개념을 어떻게 적용하는지 단계별(Step 1, 2, 3...)로 설명하세요.
4. 각 사례의 기대 결과/효과를 명시하세요.
5. 이 개념을 사용할 때 주의할 점 2~3가지를 덧붙이세요.

【답변 형식】
**활용 분야**: [어디에 사용되나?]

**사용 사례 1: [상황]**
- 시나리오: [설명]
- Step 1: ...
- Step 2: ...
- Step 3: ...
- 기대 효과: [결과]

**사용 사례 2: [상황]**
- [동일 형식]

**사용 사례 3: [상황]**
- [동일 형식]

**사용 사례 4: [상황]**
- [동일 형식]

**주의사항**:
1. 주의1: [설명]
2. 주의2: [설명]
3. 주의3: [설명]
"""
```

**검증 샘플**:
- 입력: "Python 변수란?"
- 기대 출력: 활용분야 → 4가지 사례 (각 step) → 3가지 주의

---

### 6️⃣ **관점 6: 주의사항 (Caution)**

**목표**: 이 개념 사용 시 흔한 실수, 오류, 함정 예방

**프롬프트 특징**:
- "요기 조심하세요!"에 집중
- 흔한 오류/실수 3~5개
- 각 오류의 원인 + 예방 방법
- 반대로 하면 어떻게 되는지 대조
- 길이: ~250단어

**템플릿**:
```python
PROMPT_CAUTION = """
당신은 {subject} 과목의 철저한 품질 관리자입니다.

【사용자 질문】
{question}

【검색 결과(RAG)】
{context}

【지시사항】
1. 이 개념을 사용할 때 초보자들이 자주 하는 실수 3~5가지를 나열하세요.
2. 각 실수에 대해 왜 그런 실수가 나타나는지 원인을 설명하세요.
3. 각 실수를 예방하거나 바로잡는 방법을 제시하세요.
4. ❌(잘못된 예) vs ✅(올바른 예)로 대조해서 보여주세요.
5. 이 개념의 한계나 예외 상황이 있다면 언급하세요.

【답변 형식】
**주의사항 1: [실수 제목]**
- 증상: [어떤 문제가 나타나나?]
- 원인: [왜 이런 실수가 나타나나?]
- 예방법: [어떻게 예방하나?]
- ❌ 잘못된 예: [코드/설명]
- ✅ 올바른 예: [코드/설명]

**주의사항 2: [실수 제목]**
- [동일 형식]

**주의사항 3: [실수 제목]**
- [동일 형식]

**주의사항 4: [실수 제목]**
- [동일 형식]

**주의사항 5: [실수 제목]**
- [동일 형식]

**한계 및 예외**: [한계 설명]
"""
```

**검증 샘플**:
- 입력: "Python 변수란?"
- 기대 출력: 5가지 실수 (각각 증상/원인/예방/❌vs✅) → 한계

## 📊 프롬프트 설계 가이드라인

| 관점 | 초점 | 길이 | 예시 수 | 어조 |
|-----|-----|-----|--------|-----|
| **개념** | 정의 + 배경 | ~200 | 2개 | 명확함 |
| **원리** | 단계 + 과정 | ~250 | 프로세스도 | 논리적 |
| **비유** | 관심사 + 유사점 | ~200 | 3점 비교 | 친근함 |
| **관계** | 선행/후속 + 차이 | ~250 | 비교표 | 체계적 |
| **활용** | 시나리오 + Step | ~300 | 4가지 | 실무적 |
| **주의** | 실수 + 예방 | ~250 | ❌vs✅ | 경고적 |
